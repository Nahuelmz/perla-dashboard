05 · Mejora continua

Protocolo para nuevos casos

El proceso para incorporar casos borde que aparezcan durante la beta. Cadencia semanal.

1
Guido
Detección
Un caso nuevo aparece durante el onboarding o en conversaciones reales. Guido lo documenta: mensaje del cliente, respuesta actual del bot, respuesta esperada.
2
Seba
Clasificación
Seba revisa el reporte, asigna ID, tipo y nivel de fallback. Decide si es urgente (afecta conversiones) o puede esperar al siguiente ciclo.
3
Tomi
Implementación
Tomi actualiza el system prompt o la lógica del bot. Menos de 2 horas si es de prompt. Más tiempo si requiere cambio en lógica de sistema.
4
Fede
Documentación
El caso se agrega al catálogo de esta herramienta. Se actualiza el árbol si el caso genera una rama nueva.
📅
Cadencia de revisión
Revisión semanal durante la beta. El árbol se actualiza cada vez que se agrega o modifica un caso. Los cambios urgentes (Nivel C que afecta conversiones) se implementan en el día.
